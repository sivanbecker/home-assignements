# QUESTIONS

## High-Impact Clarifying Questions

### Q1: Swagger UI route path
Where should the Swagger UI be served? Common choices: `/docs`, `/swagger`, `/api-docs`.
- **Assumption stub:** Will use `/docs` (Fastify default)

### Q2: Request ID source
Should the request ID come from an incoming header (e.g. `X-Request-Id`) or always be generated by Fastify internally?
- **Assumption stub:** Use Fastify's built-in `requestIdHeader` with `X-Request-Id`; generate if absent

### Q3: Error response shape
For non-validation errors (5xx), what JSON shape should be returned? Match Fastify's default `{ statusCode, error, message }` or a custom envelope?
- **Assumption stub:** Match Fastify's default shape to stay consistent with validation error responses

### Q4: `.env` file committed?
Should a `.env.example` be committed (with `.env` gitignored)?
- **Assumption stub:** Yes — commit `.env.example`, gitignore `.env`

### Q5: Coverage threshold values
What numeric coverage thresholds should be enforced (lines, branches)?
- **Assumption stub:** 70% lines, 60% branches — light but meaningful
