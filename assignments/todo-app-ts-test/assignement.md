## Assignment

Build a small TypeScript application for managing a list of tasks. The app should allow a user to add tasks, mark tasks as completed, remove tasks, and view the current list of tasks. Each task should have a unique identifier, a title, a completion status, and a creation date. The project should emphasize strong typing, clear data modeling, and clean code structure, which are common goals in TypeScript interview exercises. 

## Requirements

- Model the task entity with explicit types or interfaces. 
- Support creating, updating completion state, deleting, and listing tasks. 
- Prevent invalid task data, such as an empty title. 
- Keep the code organized and readable, with separation between data definitions and application logic. Clean structure and correctness are common evaluation criteria in take-home assignments. 
- Include handling for edge cases, such as trying to update or delete a task that does not exist. 
- Provide a short README with setup and run instructions. README quality is a common expectation in take-home submissions. 




## Deliverable

Submit a working project that can be installed and run locally, along with instructions for running the app and tests. Public repository delivery and runnable local setup are standard expectations for take-home tasks.
